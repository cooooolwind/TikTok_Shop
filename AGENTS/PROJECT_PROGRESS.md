# 项目进度总览

更新日期：2026-06-05

## 当前状态

项目处于核心 MVP 阶段，已经打通“素材管理 -> 脚本生成 -> 分镜视频生成 -> 预览/剪辑 -> 导出”的主链路。前端、后端、共享类型、任务队列、WebSocket 进度推送和基础部署配置均已具备可演示能力。

## 最新更新：剪辑页交互修复

- 转场预设现在可以从左侧“转场”面板拖入到底部时间线的两个片段间隙。
- 新增素材只新增片段，不再自动创建默认 `fade` 转场。
- 时间线片段之间始终保留转场投放区；已有转场块已放大，点击和选中更容易。
- 拖入“无转场”到已有转场间隙时，会删除该间隙的转场记录。
- 选中转场后，右侧属性面板现在提供显式“删除转场”按钮，删除后相邻片段直接切换。
- 进入剪辑页后默认保持空时间线，左侧素材片段仍展示可导入素材，需要用户点击“加入”或拖入后才进入剪辑轨道。
- 整体预览区域已放大，取消原先偏窄的预览宽度限制，继续保持 9:16 比例。
- 整体预览默认暂停，不再自动播放第一段；点击视频区域或按空格键才切换播放/暂停，输入框和按钮焦点下不会拦截空格。
- 拖动 playhead 时使用本地草稿位置，松手后只提交一次 Remotion seek，避免双进度线和抖动。
- 素材片段裁剪后，整体预览按源视频 in/out 区间播放，不再因转场错配或时间映射问题出现倒退重复。
- Remotion 整体预览现在只用 `frameupdate` 同步时间线，并按有效相邻转场扣除 `TransitionSeries` 重叠帧，避免完整视频播放中途被旧进度写回导致回退。
- 剪辑页整体预览已隔离 playhead 高频更新：Player 子树使用稳定 props 与 memo 保持不重建，裁剪片段使用 `trimBefore` / `trimAfter` 并启用缓冲暂停，降低播放中途回退和黑帧风险。
- 剪辑页、时间线、转场块、Remotion 预览和相关测试的中文文案已清理为正常中文。
- 本轮未新增后端接口，未修改 shared-types 契约，未新增数据库迁移。

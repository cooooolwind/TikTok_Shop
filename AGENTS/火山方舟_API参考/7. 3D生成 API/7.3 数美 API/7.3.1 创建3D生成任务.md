## 7.3.1 创建3D生成任务 

POST https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks 本文介绍创建数美 3D 生成任务 API 的输入输出参数，供您使用接口时查阅字段含义。模型会依据传入的 图片信息生成 3D，待生成完成后，您可以按条件查询任务并获取生成的 3D 文件。 

## 说明 

调用本 API 时，按输出 Token 计费，价格为 0.08 元 / 千 Token， 每输出一个 3D 模型花费 5 元至 13 元之间（视生成模式而定），详见 模型价格 。

content.**type ** string %%require%% 输入内容的类型，此处应为 text 。 

content.**text ** string %%require%% 输入给模型的文本内容。 参数（选填） ：通过 --[parameters] 的方式，控制 3D 文件输出的规格，详情见 模型文本命令(选填） 。 

callback_url string 填写本次生成任务结果的回调通知地址。当 3D 生成任务有状态变化时，方舟将向此地址推送 POST 请 求。 回调请求内容结构与 查询任务API 的返回体一致。 回调返回的 status 包括以下状态： 

- queued：排队中 

- running：任务运行中 

- succeeded： 任务成功（如发送失败，即5秒内没有接收到成功发送的信息，回调三次） 

- failed：任务失败（如发送失败，即5秒内没有接收到成功发送的信息，回调三次）

7. 3D生成 API 

**face ** integer 模型面数，支持自定义配置。取值范围为 [100000, 2000000]。 不同分辨率的推荐面数： 

- 1536： 2000000 

- 1536 pro： 2000000 

**fileformat ** integer 默认值 1 简写 ff 生成的有纹理的 3D 模型文件的格式。可选值： 

- 1 ：对应的文件格式为 obj 

- 2 ：对应的文件格式为 glb 

- 3 ：对应的文件格式为 stl 

- 4 ：对应的文件格式为 fbx 

- 5 ：对应的文件格式为 usdz 

**request_type ** integer 默认值 3 请求类型，可选值： 

- 1 ：仅生成纯几何模型 

- 3 ：一次性生成几何 + 纹理模型 

**multi_images_bit ** string 

上传的图像数组位图标识，用于标记 content.image_url 图像中各图像的视图类型。字符串由 0 和 1 组 成，1 表示该位置为有效视图图像；0 表示该位置无对应视图图像。 视图类型按顺序依次为：前视图、后视图、左视图、右视图。 示例： 

|参数取值|含义|数组内容顺序|
|---|---|---|
|1010|存在前视图、左视图|[前视图, 左视图]|
|1011|存在前视图、左视图、右视图|[前视图, 左视图, 右视图]|
|1111|四个视图均存在|[前视图, 后视图, 左视图, 右视<br>图]|
|0101|存在后视图、右视图|[后视图, 右视图]|

## 7.3.2 查询3D生成任务 

GET https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks/{id} 查询 3D 生成任务的状态。 

## 说明 

仅支持查询最近 7 天的历史数据。时间计算统一采用 UTC 时间戳，返回的 7 天历史数据范围以用 户实际发起查询请求的时刻为基准（精确到秒），时间戳区间为 [T-7天, T)。

## 14.8.4 GetRecordExportTask - 查询用量明细导出任务状 

## 态 

查询用量明细导出任务状态，并获取文件下载链接。 

## 说明 

- 需传入通过 CreateRecordExportTask 接口获取的任务 ID 作为入参。 

- CreateRecordExportTask 任务提交后，建议等待 5 秒再发起 GetRecordExportTask 轮询： 

- ◦ 轮询间隔：不小于 5 秒 

- ◦ 单任务轮询总时长：不超过 5 分钟 

   - 当接口返回 Succeeded 状态且包含 DownloadUrl 字段时，请立即下载 CSV 文件，避免链接 过期失效，链接有效期为 24 小时。

Status string 示例值：Running 任务状态机，取值：

ExportCount long 示例值：5000 实际写入 CSV 文件的行数，最大不超过 100000；当 TotalCount > ExportCount 时，表示导出结果因达到 单任务行数上限而被截断。

## 注意 

下载链接有效期为 24 小时。链接过期后， FileExpired 字段值为 true ，需重新提交任务获取新 链接。 

DownloadUrlExpireTime long 示例值：1762041600 下载链接的过期时间（UTC Unix 时间戳，单位：秒），有效期 24 小时，过期后对应 TOS 文件将被清理。 

FileExpired float 示例值：false 文件是否已过期标识，当当前时间大于等于 DownloadUrlExpireTime 时为 true 。 

ErrorMessage string 任务失败时的指引文案，仅当任务状态为 Failed 时存在有效值。 

CreateTime long 示例值：1761955200 任务创建时间（UTC Unix 时间戳，单位：秒）。 

StartTime long 示例值：1761955260 任务开始时间（UTC Unix 时间戳，单位：秒），对应任务状态切换为 Running 的时刻。任务未调度时该 字段不返回；若任务触发系统重试，此字段始终保留首次启动时间。 

EndTime long 示例值：1761955320 任务结束时间（UTC Unix 时间戳，单位：秒）。代表任务进入 Succeeded 、 Failed 终态的时间；任务处 理中时，该字段不返回。
